iOS小知识点备忘

Posted on 2018-05-24

1.NSLocalizedString 和 NSLocalizedStringFromTable 区别

①如果在命名.strings文件为Localizable.strings时，那么你就可以用以下方式调用：
NSLocalizedString(@”name”, nil)
NSLocalizedStringFromTable(@”str0”, @”Localizable”, nil))
两者功能一致。
②如果创建.strings文件是自定义的名字的话，如当前我起名为Internation.strings,那么调用的方式为：
NSLocalizedStringFromTable(@”name”,@”Internation”, nil)
当自定义名字后，使用NSLocalizedString的方式不起作用。

property常用修饰符

①多线程特性修饰符
atomic：原子的。表示线程安全，编译器自动生成互斥加锁代码，但不保证一定安全且浪费系统资源。
nonatomic：非原子的。表示非线程安全，编译器会少生成一些互斥加锁代码，提高效率。

建议：涉及多线程，使用atomic，保证安全。不涉及多线程，使用nonatomic,，效率高。

②读写特性
readwrite：默认特性。编译器为属性生成getter和setter方法。
readonly：编译器只生成getter方法。仅供外界读取，在内部修改为readwrite属性。

③内存管理特性
MRC时期：
assign：赋值。第一用于基本数据类型和C数据类型；第二用于id类型，避免强引用循环，但对象销毁时，指针不会被置空，可能会引起空指针。
retain：持有。先release原来的值，在retain新值，引用计数自动+1。

ARC时期：(iOS5以后开始支持ARC管理内存)
strong：默认特性。对象引用计数+1，是指针拷贝（浅拷贝），取代MRC时期的retain。
weak：表示“非拥有关系”，引用计数不+1，对象销毁时，指针自动被置为nil，放置野指针。取代MRC时期的assign的第二功能。
unsafe_unretained：取代assign的第一功能。
copy：复制一个相同对象并创建strong指针，是内容拷贝（深拷贝）。

Hexo+Github Pages搭建个人博客

Posted on 2018-05-09

折腾了两天，对如何使用Hexo+Github Pages搭建博客有了一些理解，现在记录下来，方便自己以后翻阅使用。

工具简介

1、Hexo
Hexo是一个快速、简洁且高效的博客框架，在这里可以了解更多。
2、Git、GitHub、GitHub Pages三者的关系
Git：是Linux的创始人Linus在2005年用C编写的一个分布式版本控制系统，详细请看Git简史。
GitHub：是一个共享虚拟主机服务，用于存放使用Git版本控制的软件代码和内容项目。
GitHub Pages：GitHub提供给开发者来存放静态网页，展示作者的想法或项目说明。
3、GitHub Pages类型
· User/Organization Pages（个人或公司站点）
①这种类型的Pages通过新建一个仓库建立，以自己的用户名命名，且每个用户名只能建立 一个；
②仓库主干（master）上内容被用来构建和发布页面
③多用于建立个人博客（这正是本篇的目的所在）
· Project Pages 项目站点
①这种类型的仓库通过修改项目的Setting建立，一个项目只能建立一个；
②仓库新建了一个gh-pages分支用于构建和发布页面；
③多用于展示项目效果、给出项目说明、分享作者思想等

搭建博客原理

所谓搭建个人博客，通俗来讲就是使用Hexo这一个博客框架来书写博客内容生成HTML文件部署到可以展示HTML文件的服务器上（如GitHub的Pages服务，Coding的Pages服务等）。

搭建过程

1、注册GitHub账号
关于如何创建GitHub账号就不必多说了，去官网一目了然。
2、创建仓库
①创建 用户名.github.io 仓库（必须使用用户名）。
②使用用户名创建仓库，默认开始GitHub Pages服务。
③博客可以通过仓库名直接访问，如果不使用用户名，则不能访问。
3、安装Git
Mac一般自带Git，如果没有可至官网下载一路安装即可。
4、安装Node.js
下载Node.js
安装Node.js
如果已安装Node.js，可使用下面操作进行版本更新
①查看本机版本

node -v

②清除node.js的cache

sudo npm cache clean -f

③安装n工具，这个工具是专门用来管理node.js版本的

sudo npm install -g n

④安装最新版本的node.js：

sudo n stable #最新稳定版本（建议安装）
sudo n latest #最新版本

⑤使用①中命令查看版本
5、安装Hexo
①在任意处（我选择桌面）创建Hexo目录，以命令行方式打开文件夹
②执行如下命令

npm install hexo-cli -g #安装Hexo
hexo init [文件夹名] #初始化网站
npm install #作用是按照package.json安装所需要的组件放在生成的node_modules文件夹中
hexo g #生成静态页面至public目录
hexo s #启动本地服务器，可以通过http://localhost:4000查看

一些常用hexo命令

hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本
hexo deploy -g #生成加部署
hexo server -g #生成加预览
命令的简写
hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy

6、部署到GitHub
①修改_config.yml文件
deploy:
type: git
repo: git@github.用户名/用户名.github.io.git #这里的网址填你自己的
branch: master

②安装一个拓展

npm install hexo-deployer-git –save

③开始部署

hexo d

添加/更换主题

1、安装主题

hexo clean
git clone 主题Git地址 themes/主题名，下面以NexT主题为例
git clone https://github.com/iissnan/hexo-theme-next themes/next

2、启动主题
Hexo根目录下找到_config.yml文件，打开找到theme:属性并设置为主题名，必须与1中主题名一致。

3、更新主题

cd themes/主题名
git pull

4、生成启动
回到Hexo根目录下执行

hexo g
hexo s
